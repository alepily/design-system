About these insights Our service calls each bank regurlarly, then evaluates the returned payload against a number of criteria. %: the availability represents the percentage of time (over the period of testing) for which the API platform was responding correctly - where data was returned i a manner conformant to the API specification imolemented by the ASPSP in question. In certain cases, no data nay be shown (-). which indicates that no active consent was available at the time an attempt was made to cakk APSP protected resources. ms: The performance metric is a representation of the average time in milliseconds to get a response. This is an aggregate view across all of the protected resources offered by the ASPSP, and is calculated as a weighted average. Yapily Score (YS): This is calculated by evaluating the response time and responses sizes of valid requests for accounts and transactions over the last week. Using a combination of these metrics, Yapily scores each bank according to how well the banks perform comparatively.
