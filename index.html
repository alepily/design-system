Goals and principles of this design system

The design system should...

  • Speed to build any frontend required
  • Create consistency in all UIs.
  • Provide re-usable components.
  • Be accesible and usable!
  • Contain good practices in writing the code, semantic.
  • Look great on mobile, desktop and huge screens (fill the space in a smart way) without affecting the UX.
  • Be easy to understand and implement for anyone joining in the future.
  • Be well documented how to use and how to implement.
  • Be able to adapt to branding changes it quickly.
  • Be able to be re-skinned easily.

Typography

Length for content should be max 100 characters, left aligned. In the exception of the content being aligned at the center, align also the typography. All font sizes have a ratio of 1.125, known as the major second type scale with a base size of 16px. This means that each size is multiplied or divided by 1.125 from the previous size.

Forms

Forms allow people sending multiple information in a simple way, we should only ask what we need.

  • Primary buttons should be place on the right.
  • Multi-column layouts should be avoided.
  • When content is centered use the full width for inputs and primary button
  • Mark optional fields with (optional), don't mark mandatory fields. Always use a label on top.
